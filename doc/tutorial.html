gim tutorial

In this tutorial we will start a new NONMEM/PsN modeling project with gim from scratch. The tutorial assumes you have gim and git installed, using the installation guide.

Initialization

First, we'll create a folder and initialize gim. Choose your foldername wisely as gim assumes it is the same as the name of your git repository in the cloud.

cd ~/projects
mkdir proj_x
cd proj_x

Next, we will need to create the Git repository and hook it up to a git cloud service such as GitHub or BitBucket. For this, go to the respective website (e.g. github.com) and create a new repository, with the same name as you used for the folder name (e.g. proj_x). We then initiate our git repository on the local machine and link it to the cloud. Assuming you use BitBucket for your project:

gim init bitbucket

or for GitHub:

gim init github

Note: Other git servers can be defined in your setup file (~/.gim/settings.json).

You should see this in your console:

gim: creating new git repository in current folder.
gim: done.

Now you are set to go. You can add models and datasets to the folder and start modeling.
